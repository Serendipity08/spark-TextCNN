# 情绪礼物 V1 两周开发计划书

时间范围：2 周（10 个工作日）  
目标：完成 Web 端闭环体验与可分享礼物的完整链路，具备可演示与可小范围试用能力。  

## 1. 目标与范围
本期必须完成（V1）：
- 创作端：文字/语音输入、AI 轻润色、场景选择、生成礼物
- 接收端：礼物展示、播放体验、回执反馈
- 生成后可预览、可修改文案/场景并重新生成
- 后端接口可用（polish/tracks/gifts/ack/like）

本期暂不做（V2）：
- 语音音色克隆与 TTS
- 自动混音合成
- 账号体系/支付

## 2. 技术交付物
前端（Web/H5）：
- 创作端页面 + 接收端页面 + 预览弹层
- 场景模块优化（描述/视觉/试听）
- API 集成与错误提示

后端（Spring Boot）：
- API v1 完整实现（polish/tracks/gifts/ack/like）
- 礼物状态流转与存储（内存 or MySQL）
- 运营基础数据（曲库/场景配置）

## 3. 分阶段计划（10 天）

### 第 1 周：核心闭环
Day 1（范围锁定与设计）  
- 确认 V1 功能边界与交互细节  
- 设计预览体验（生成后可修改）  
产出：V1 交互说明、场景模块结构

Day 2（前端结构搭建）  
- 创作端 UI 调整（场景卡片、润色候选区）  
- 接收端 UI 状态完善（加载/过期/错误）  
产出：前端页面结构与组件样式

Day 3（后端接口打通）  
- `/polish` 接入真实 AI 或可替换的 Mock  
- `/tracks` 返回场景 + 试听片段 + startAt  
产出：接口可调通，前端联调

Day 4（生成与预览机制）  
- 创作端生成礼物后进入预览层  
- 预览支持“换场景 / 重新润色 / 重新生成”  
产出：完整的“生成 → 预览 → 确认”闭环

Day 5（基础接收链路）  
- 接收端读取礼物详情  
- 回执接口联调（ack/like）  
产出：可分享、可接收、可回执

### 第 2 周：稳定与优化
Day 6（场景体验优化）  
- 场景卡片增加描述、氛围标签  
- 音乐试听逻辑与视觉预览关联  
产出：场景模块体验升级

Day 7（容错与日志）  
- 前端错误提示（网络/接口失败）  
- 后端日志与异常响应规范  
产出：稳定性与可诊断性提升

Day 8（数据存储/配置）  
- 落地存储（内存 → MySQL 可选）  
- 曲库配置可维护  
产出：可长期运行的数据基础

Day 9（联调与自测）  
- 端到端完整流程测试  
- 关键指标埋点（生成/打开/完播/回执）  
产出：可演示版本

Day 10（验收与发布准备）  
- 体验验收、问题修复  
- 准备部署说明与演示脚本  
产出：可对外演示版本

## 4. 关键里程碑
- M1（第 5 天）：闭环可用（创作→预览→分享→接收→回执）
- M2（第 10 天）：体验优化与稳定性达标，可演示

## 5. 风险与依赖
依赖：
- AI 服务可用（文案润色）
- 曲库与试听资源

风险：
- AI 输出不稳定 → 增加“保留原文/重新生成”
- 音乐资源版权 → 确保使用授权或可商用素材

## 6. 验收标准
- 创作端完成文字/语音输入、润色、场景选择、生成
- 生成后可预览与调整
- 接收端可打开、展示、播放与回执
- API 调用稳定、错误提示友好
